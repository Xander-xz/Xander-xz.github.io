小白实现双向链表需要注意的事项

直接上代码：
#include <stdio.h>

#include <stdlib.h>

// 定义双向链表节点结构

struct Node {

    int data; // 节点数据

    struct Node* next; // 指向下一个节点的指针

    struct Node* prev; // 指向上一个节点的指针

};

// 创建一个新节点

struct Node* createNode(int data) {

    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node)); // 动态分配内存

    newNode->data = data;     // 设置节点数据

    newNode->next = NULL;     // 初始化next指针为NULL

    newNode->prev = NULL;     // 初始化prev指针为NULL

    return newNode;           // 返回新节点

}

// 向链表末尾添加节点

void append(struct Node** head_ref, int new_data) {

    struct Node* new_node = createNode(new_data); // 创建新节点

    struct Node* last = *head_ref; // 初始化last节点为头节点

    if (*head_ref == NULL) { // 如果链表为空

        *head_ref = new_node; // 新节点为头节点

        return;

    }

    while (last->next != NULL) // 遍历到链表末尾

        last = last->next;

    last->next = new_node; // 将新节点添加到链表末尾

    new_node->prev = last; // 设置新节点的prev指针

}

// 打印链表

void printList(struct Node* node) {

    struct Node* last;

    printf(“Traversal in forward direction:\n”);

    while (node != NULL) { // 从头到尾遍历链表

        printf(“%d “, node->data);

        last = node; // 记录当前节点

        node = node->next; // 移动到下一个节点

    }

    printf(“\n”);

}

int main() {

    struct Node* head = NULL; // 初始化头节点为NULL

    append(&head, 1);

    append(&head, 2);

    append(&head, 3);

    append(&head, 4);

    printList(head); // 打印链表

    return 0;

}

为什么添加节点一定要用二级指针，不能用一级指针？
我个人的理解是：head作为头结点，它也是指针，指针存放的就是地址，当然它本身也有地址，如果你不把指针的地址传给二级指针的话，那么每次调用append这个函数的时候，那么head永远给append的函数传的NULL，你永远都改变不了head的值，“&”这个是地址运算符，只有&head才是取head的地址，而如果传一个head给函数，那么只是把head指向的内容告诉了你。

这个结点存放的是一个数据域和两个指针域，数据域才是用户需要的数据，指针域是指向上一个结点或下一个结点，链表一定是从头结点开始的，如果你没有头结点，你就会找不到这个链表，逻辑上也是说不通的。

有人说c语言不是高级语言是中级语言，我觉得这个是说不通的，众所周知Java是面向对象的语言，而c语言是面向过程的语言，不过c语言其实也是有对象，而且很多功能都可以实现，c语言的对象，叫做数据对象，数据对象可以静态实现和动态实现，我们这个例子中用的是malloc动态链表，调用对象不就是调用地址嘛，所以这里的对象很显然就是head结点开始，不要被结点这种名字给弄晕了，链表作为数据结构，它把对象都存在到了结点里，结点是怎么定义的，不就是结构体嘛，由结构体产生对象，然后每一个结点存放下一个结点的地址，以及下一个结点存放前一个结点的地址，然后就形成了双向链表，数据元素指的是数据对象内部的具体值，在这个代码中，p->data就是数据元素，数据项是最小单位，数据项就是对象中的一个字段或属性，在个代码中就是data，像那些出书的人总是喜欢玩文字游戏，所谓的面向过程，这个过程其实说的就是函数，你看看上面的代码，结点的创建是用结构体，对象的创建是用的函数，对象的空间是用malloc函数开辟，有感受到割裂感了嘛，面向对象就不那么做了，直接把属性和函数全部放在一起，形成一个新的数据结构，就是“类”，以类为模版创建对象，Java是纯面向对象的语言，但又不是那么纯，毕竟Java有自动装箱和拆箱的功能，这个是编译器自动处理的，面向对象对事物的描述更加准确，我定义一个结点类，就能就能实现所见即所得了吧，面向过程的话，它把数据的定义和函数给分开了，还要说一句的是万事万物皆是数据，我们说的数据，和你想的可能是不一样的，我们说的数据那个范围是非常广的，不仅限于数字123，数据结构不正是存放数据的容器嘛，而链表和顺序表是最基础的，是一对一的关系，通过这个数据结构，我们可以推广出更复杂的数据结构。
上面的代码量非常少，只是创建了链表，然后插入了之后，再显示出来，是非常简答的，只是有的人C语言不好的人，看起来会有压力，多看几遍就行了，比如像这个函数定义“struct Node createNode(int data) ”，是不是看起来很奇怪，它其实是定义了一个struct Node类型的函数，函数名叫做createNode，参数是int data，即返回值是一个指针，这个需要和指针变量区分一下，指针变量的话，可以加个括号，函数指针变量就是接收其他函数的地址，然后实现钩子函数，就是类似面向对象语言中的多态，就是不同的对象，会产生不同的行为，也就产生了多态，钩子函数也可以叫事物处理函数，就是你传什么函数的指针，我就执行什么，还有编译器的话其实只认参数的声明，参数名其实并不是最重要，主要是给我们人看的，现在不是流行语义化了嘛，我想就是这个道理，暂时先说这么多吧。
